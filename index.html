GitHub Cloudflare Telegram Tool

GitHub 設定

1

取得 GitHub ID

GitHub ID 是你的 GitHub 使用者名稱（Username）。可從個人頁面的網址取得，例如 https://github.com/octocat，其中 octocat 即為 GitHub ID。

  • 登入 GitHub 後，點擊右上角頭像 → 進入「Your profile」。
  • 查看瀏覽器網址列，網址最後一段即為 GitHub ID。
  • GitHub ID 欄位只填使用者名稱本身（例如：octocat）。
  • 請勿填寫完整網址、Email 或顯示名稱（Name）。
2

填寫 Repo name 並選擇建立方式

請輸入你要使用的 Repo name，並選擇要「自行建立」或「交由系統自動建立」。

  • 自行建立：請參考步驟 3 說明。
  • 系統自動建立：勾選後可直接跳至步驟 4 建立 GitHub PAT。

兩種方式都可以，但此處填寫的 Repo name 必須與最後實際要使用的名稱完全一致。

3

自行建立 GitHub Repo

請前往 GitHub 建立 Repo 頁面，依照下方畫面完成設定。

  • Repository name 必須與表單填寫的名稱完全一致。
  • Visibility 務必設為「Private」，其他選項保持關閉。

GitHub Repo 建立圖示 1

確認 Repo 名稱與 Visibility 私有化設定。

GitHub Repo 建立圖示 2

點擊最下方按鈕完成建立。

4

建立 GitHub PAT

開啟 Fine-grained PAT 建立頁面（已帶入必要權限），依照下方畫面完成設定後，將產生的 token 複製回此表單。

  • Repository access 請選擇「Only select repositories」並選取對應 Repo。
  • 此連結已帶入必要權限與設定，請勿自行修改（特別是 Permissions 與 Repo access），否則可能導致後續自動化流程失敗。

PAT 建立頁面

確認 Resource owner（帳號）、Repo access 與 Permissions 設定皆正確。

建立後複製 token

建立後僅顯示一次 token，需立即複製保存。

1

取得 Account ID

進入 Cloudflare Dashboard，切換到本次要部署的帳號，從右上角選單取得 Account ID。

  • 多帳號時請確認已切換到正確的 account。

Account ID

從右上角選單點擊「Copy account ID」取得帳號識別碼。

2

建立 Cloudflare API Token

進入 Cloudflare API Tokens 頁面，依照下方畫面建立 Token。

  • 請使用「Edit Cloudflare Workers」模板作為起點。
  • 在 Permissions 區塊按下「Add more」，額外加入 D1 權限後再繼續。
  • 確認 Token 綁定的 account 與前一步的 Account ID 一致。

⚠️ 請務必使用「Edit Cloudflare Workers」模板，並在 Permissions 區塊另外補上 D1 權限。若權限不足或設定錯誤，會導致部署失敗。

API Token 列表頁

進入 API Tokens 頁面，點擊「Create Token」。

選擇 Workers 模板

選擇「Edit Cloudflare Workers」，點擊「Use template」。

權限設定

保留 Workers 預設權限，並按下「Add more」額外加入 D1 權限。

Account / 資源設定

在 Account Resources 選擇本次使用的 account。

建立 Token

點擊「Continue to summary」，再按「Create Token」。

複製 Token

Token 只會顯示一次，請立即複製並保存。

3

Workers 初始化（首次使用才需要）

如果你是第一次使用 Cloudflare Workers，請先進入 Workers & Pages 頁面完成初始化。

若已看到 subdomain，則代表已完成初始化，可直接跳過此步驟。

Workers 初始化畫面

進入 Workers & Pages 後會自動初始化。

1

建立 Telegram Bot 並取得 Token

透過 BotFather 建立 bot，並在建立完成後立即取得 Bot Token。

  • bot username 必須以 bot 結尾（例如：my_claw_bot）。
  • Bot Token 只會顯示一次，請立即複製，切勿公開分享。

搜尋 BotFather

在 Telegram 搜尋「BotFather」。

建立 Bot

點擊「Create a New Bot」。

填寫 Bot 資訊

輸入 bot 名稱與 username。

取得 Token

建立完成後，點擊「Copy」取得 Bot Token。

2

讓 bot 收到訊息並取得 Chat ID

開啟你剛建立的 bot 對話，送出第一則訊息後，回到表單點擊「自動取得 Chat ID」。

GitHubClaw 是透過 Telegram 的訊息更新紀錄取得 Chat ID。

開啟 bot 資訊頁

點擊 bot 的 username。

開啟 bot 對話

在 bot 資訊頁中點擊「Message」。

傳送第一則訊息

第一次進入聊天室時，點擊「START」。

確認 bot 已收到訊息

只要聊天室中已經有你送出的訊息即可。

自動取得 Chat ID

回到 GitHubClaw 點擊「自動取得 Chat ID」。
