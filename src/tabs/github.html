階段一：建立基地與機密 (Infrastructure & Secrets)

這是小龍蝦運行的舞台。我們將在 GitHub 建立倉庫並設定存取權限，這是所有工作的基礎。

GitHub 使用者資訊：

確認你的身分證，讓系統知道誰是「大海」的主人。

建立 GitHub 倉庫 (Repository)：

這是一切任務發生的空間，也是小龍蝦的「家」。

個人存取權杖 (PAT)：

這是小龍蝦代你執行任務的合法授權，請妥善保管。

⚠️ 注意事項

Token 安全：

絕對不要洩漏你的 PAT 或 Secrets。一旦 Commit 到公開區域，你的帳號權限可能受損。

最小權限：

設定 PAT 時，只需選擇必要的 Repositories。不需要給全域存取權限，以保障其他 Repo 的安全。

1

取得 GitHub ID

GitHub ID 是你的 GitHub 使用者名稱（Username）。可從個人頁面的網址取得，例如 https://github.com/octocat，其中 octocat 即為 GitHub ID。

  • 登入 GitHub 後，點擊右上角頭像 → 進入「Your profile」。
  • 查看瀏覽器網址列，網址最後一段即為 GitHub ID。
  • GitHub ID 欄位只填使用者名稱本身（例如：octocat）。
  • 請勿填寫完整網址、Email 或顯示名稱（Name）。
2

填寫 Repo name 並選擇建立方式

請輸入你要使用的 Repo name，並選擇要「自行建立」或「交由系統自動建立」。

  • 自行建立：請參考步驟 3 說明。
  • 系統自動建立：勾選後可直接跳至步驟 4 建立 GitHub PAT。

兩種方式都可以，但此處填寫的 Repo name 必須與最後實際要使用的名稱完全一致。

3

自行建立 GitHub Repo

請前往 GitHub 建立 Repo 頁面，依照下方畫面完成設定。

  • Repository name 必須與表單填寫的名稱完全一致。
  • Visibility 務必設為「Private」，其他選項保持關閉。

4

建立 GitHub PAT

開啟 Fine-grained PAT 建立頁面（已帶入必要權限），依照下方畫面完成設定後，將產生的 token 複製回此表單。

  • Repository access 請選擇「Only select repositories」並選取對應 Repo。
  • 此連結已帶入必要權限與設定，請勿自行修改（特別是 Permissions 與 Repo access），否則可能導致後續自動化流程失敗。

PAT 建立頁面

確認 Resource owner（帳號）、Repo access 與 Permissions 設定皆正確。
