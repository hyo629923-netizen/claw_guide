階段四：建立大腦與演進 (The AI Brain & Evolution)

這是賦予小龍蝦智慧與持續演進的關鍵。除了將你的「白話文」轉化為具體工作，大腦還具備自動更新與環境重置的能力，確保系統始終保持在最新狀態並及時反饋。

任務中控 (GitHub Issues)：

大腦將分析後的任務寫入 Issue，作為指令傳遞的橋樑。

智慧路由 (AI Brain Logic)：

透過提示工程 (Prompt Engineering)，讓 AI 決定該叫哪一隻小龍蝦工具出動。

核心自動更新與回傳：

透過指令觸發核心引擎的安全更新與環境同步，確保工作流程的準確性與最新進度反饋。

⚠️ 注意事項

任務標籤：

大腦會幫任務打上標籤（如 爬蟲），讓特定小龍蝦工人能即時「認領」屬於它的工作。

GitHub Issues：任務的指令中控室

在小龍蝦的生態系中，GitHub Issue 不只是用來紀錄 Bug，它是 **「通訊軟體」與「自動化工人工坊」** 之間的橋樑。

1

任務轉換

當你對著 Telegram 說話，AI 大腦會分析需求，並透過 GitHub API 在倉庫中建立一則特定的 Issue。

2

指令封裝

  • Issue 標題： 定義「哪隻小龍蝦」工作（例如：[爬蟲] 任務名稱）。
  • Issue 內容： 提供 AI 整理後的結構化參數（如 URL 或關鍵字）。
  • Issue 標籤： 用來過濾與追蹤進度。

指令執行的生命週期 (Lifecycle)

了解一個指令如何從小龍蝦的大腦出發，最終完成任務並回到你的手中。

1

語意解析 (Thinking)

接收 Telegram 指令後，AI 會對讀取現有的 .memory 進行脈絡比對。

2

任務封裝 (Packaging)

將指令轉化為包含參數與環境設定的 GitHub Issue。

3

自動執行 (Execution)

GitHub Actions worker 根據 Issue 標籤自動認領並動手做事。

4

成果回饋 (Reporting)

執行完成後，結果與檔案會直接留言在 Issue 並推送到 Telegram。

智慧記憶與脈絡 (.memory)

大腦之所以聰明，是因為它擁有「長期記憶」。所有執行過的任務、學習到的知識與使用者的習慣，都會沉澱在 .memory 資料夾中。

動態知識 (Dynamic)

每完成一項任務，大腦都會總結經驗並更新記憶，減少下次重複學習的時間。

跨工作區共享 (Shared)

不同小龍蝦工人可以讀取共享記憶檔案，確保整個生態系的步調一致。

GitHub Actions：全天候的執行工坊

GitHub Actions 是真正的「小龍蝦工人」，它們在後端安靜且快速地處理各種繁重任務。

自動化觸發流程

on:
issues:
types: [opened]
jobs:
run-lobster:
# 檢查標題，決定是否動工
if: contains(github.event.issue.title, '爬蟲')

工人的工作特點：

無須喚醒

Issue 一開，Action 立即由 GitHub 分配機器執行。

狀態自動回傳

執行結束後，自動將結果留言在 Issue 下方，並傳回通訊軟體。

小龍蝦的大腦並非一成不變，它在與你的每次互動中不斷學習與演進。
