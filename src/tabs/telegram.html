階段二：建立通訊與觸發 (ChatOps & Trigger)

讓你的通訊軟體成為指令發射站。我們將建立 Telegram Bot 並取得必要的 ID，讓小龍蝦能接收你的命令。

建立 Telegram Bot：

向 @BotFather 申請一個專屬機器人，並獲取 API Token。

取得專屬 Chat ID：

這像是一個私人的門牌號碼，確保機器人只回覆你，不理會陌生人的騷擾。

⚠️ 注意事項

身分驗證 (Whitelisting)：

取得 Chat ID 是為了安全。如果你不做 User ID 檢查，任何對機器人說話的人都能控制你的小龍蝦執行任務。

機器人與 GitHub 連動：

一旦設定完成，你在 Telegram 送出的指令將會透過 Webhook 或轉發機制，最終變成 GitHub 上的 Issue。

1

建立 Telegram Bot 並取得 Token

前往 @BotFather 建立你的機器人，並獲取 API Token。

  • 傳送 /newbot 給 BotFather。
  • 設定機器人的顯示名稱與 username。
  • 完成後 BotFather 會給出一串 API Token（例如：123456:ABC-DEF...）。

BotFather 建立 Token

複製並保存 Token 到表單中。

2

讓 bot 收到訊息並取得 Chat ID

開啟你剛建立的 bot 對話，送出第一則訊息後，回到表單點擊「自動取得 Chat ID」。

GitHubClaw 是透過 Telegram 的訊息更新紀錄取得 Chat ID。

開啟 bot 資訊頁

點擊 bot 的 username。

開啟 bot 對話

在 bot 資訊頁中點擊「Message」。

傳送第一則訊息

第一次進入聊天室時，點擊「START」。

確認 bot 已收到訊息

只要聊天室中已經有你送出的訊息即可。

自動取得 Chat ID

回到 GitHubClaw 點擊「自動取得 Chat ID」。
