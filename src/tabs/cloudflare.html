階段三：部署 Webhook 中繼站 (Middleware Setup)

這是連接通訊軟體與 GitHub 的橋樑。我們使用 Cloudflare Workers 作為中繼，處理身分驗證、自動過濾垃圾訊息，並將合法指令轉發給 GitHub 大腦。

Cloudflare 帳號資訊：

取得你的 Account ID，這是識別你邊緣運算空間的唯一代碼。

API Token 與 權限設定：

透過專屬的 API 權杖授權系統自動部署 Workers 程式碼，需同時具備 Workers 與 D1 資料庫權限。

邊緣運算初始化 (Workers Init)：

啟動你的雲端空間，準備迎接 24 小時不間斷的中繼轉發服務。

⚠️ 注意事項

角色定位：

Cloudflare 不負擔沉重的計算，它只負責「聽」指令、「過濾」不法使用者，然後「通知」GitHub 小龍蝦。它就像是一個專業的保全與接待處。

1

取得 Account ID

進入 Cloudflare Dashboard，切換到本次要部署的帳號，從右上角選單取得 Account ID。

  • 多帳號時請確認已切換到正確的 account。

Account ID

從右上角選單點擊「Copy account ID」取得帳號識別碼。

2

建立 Cloudflare API Token

進入 Cloudflare API Tokens 頁面，依照下方畫面建立 Token。

  • 請使用「Edit Cloudflare Workers」模板作為起點。
  • 在 Permissions 區塊按下「Add more」，額外加入 D1 權限後再繼續。
  • 確認 Token 綁定的 account 與前一步的 Account ID 一致。

⚠️ 請務必使用「Edit Cloudflare Workers」模板，並在 Permissions 區塊另外補上 D1 權限。若權限不足或設定錯誤，會導致部署失敗。

API Token 列表頁

進入 API Tokens 頁面，點擊「Create Token」。

選擇 Workers 模板

選擇「Edit Cloudflare Workers」，點擊「Use template」。

權限設定

保留 Workers 預設權限，並按下「Add more」額外加入 D1 權限。

Account / 資源設定

在 Account Resources 選擇本次使用的 account。

建立 Token

點擊「Continue to summary」，再按「Create Token」。

複製 Token

Token 只會顯示一次，請立即複製並保存。

3

Workers 初始化

如果你是第一次使用 Cloudflare Workers，請先進入 Workers & Pages 頁面完成初始化。

若已看到 subdomain，則代表已完成初始化，可直接跳過此步驟。

Workers 初始化畫面

進入 Workers & Pages 後會自動初始化。
